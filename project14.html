Web Socket

Web Socket and Ajax

Github Repository Link


Functionality

The project involves connecting a web browser to a webservice using Websockets, without using XML. JSON is recommended. The aim is to make a shared drawing program work using Websockets, allowing multiple clients to connect and draw on the same surface and share their drawings with others. The project also aims to teach the basics of Websockets, canvas, JSON, and the Flask micro-framework, and to compare using Websockets versus AJAX. User stories include drawing on the canvas, watching others draw, and the system being responsive to actions. The webservice must pass tests, serve /static/index.html, and display current state of the webservice. The final product should look nice and be functional. Collaborating with others is allowed, but external source code must be documented.
